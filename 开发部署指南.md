## 二、环境搭建

### 2.1 安装 DevEco Studio

1. **下载地址**: [HarmonyOS DevEco Studio](https://developer.huawei.com/consumer/cn/deveco-studio/)
2. **安装步骤**:
   - 运行安装包，选择安装路径
   - 安装完成后启动，进入配置向导
   - 安装鸿蒙 SDK（API 26 及以上）

### 2.2 配置 SDK

1. 打开 DevEco Studio → Settings → Appearance & Behavior → System Settings → HarmonyOS SDK
2. 选择 SDK Platforms 标签
3. 勾选 `API Version 26` 及相关工具
4. 点击 Apply 进行安装

### 2.3 配置模拟器

1. 打开 Device Manager（工具栏图标）
2. 点击 Create Virtual Device
3. 选择 Phone → HarmonyOS NEXT Phone
4. 点击 Finish 创建模拟器

## 四、后端服务部署

### 4.1 获取后端代码

```bash
git clone <后端仓库地址> kinvoice-backend
cd kinvoice-backend
```

### 4.2 安装依赖

```bash
pip install -r requirements.txt
```

### 4.3 配置环境变量

创建 `.env` 文件：

```env
# 服务器配置
HOST=0.0.0.0
PORT=8000

# 数据库配置
DATABASE_URL=sqlite:///./kinvoice.db

# AI 服务配置
LLM_API_KEY=<你的API密钥>
LLM_BASE_URL=https://api.openai.com/v1

# 跨域配置 - 必须包含鸿蒙应用的请求来源
CORS_ORIGINS=http://localhost:8000,http://127.0.0.1:8000,http://localhost,http://127.0.0.1
```

### 4.3.1 CORS 配置说明

**HarmonyOS 应用跨域要求**:

由于鸿蒙应用使用 `@ohos.net.http` 发起 HTTP 请求，后端必须正确配置 CORS（跨域资源共享）。

**FastAPI 后端配置示例**:

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# 允许的来源 - 必须包含鸿蒙应用可能的请求来源
origins = [
    "http://localhost:8000",
    "http://127.0.0.1:8000",
    "http://localhost",
    "http://127.0.0.1",
    # 如果使用真机调试，添加你的电脑 IP
    "http://192.168.1.100:8000",
]

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**关键配置项**:

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `allow_origins` | `["*"]` 或具体域名列表 | 允许的请求来源，建议使用具体列表 |
| `allow_credentials` | `True` | 是否允许携带 Cookie/Token |
| `allow_methods` | `["*"]` | 允许的 HTTP 方法 |
| `allow_headers` | `["*"]` | 允许的请求头，必须包含 `Authorization` |

**调试建议**:

1. **模拟器调试**: 使用 `localhost` 或 `127.0.0.1`
2. **真机调试**: 使用开发电脑的局域网 IP（如 `192.168.1.100`）
3. **生产环境**: 配置具体的域名列表，禁止使用 `["*"]`

### 4.4 启动服务

```bash
# 开发模式（带热重载）
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 生产模式
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 4.5 验证服务

访问 `http://localhost:8000/docs` 查看 API 文档（Swagger UI）

## 七、常见问题

### 7.1 网络连接失败

**问题**: 应用无法连接到后端服务

**解决方案**:
1. 确认后端服务正在运行
2. 检查防火墙设置，允许 8000 端口
3. 使用 IP 地址而非 localhost（模拟器需要）

### 7.2 CORS 错误

**问题**: 跨域请求被阻止

**解决方案**:
1. 确认后端已配置 CORS
2. 使用代理服务器

### 7.3 构建失败

**问题**: hvigorw 构建报错

**解决方案**:
1. 检查 Node.js 版本
2. 清除构建缓存：`hvigorw clean`
3. 更新依赖：`hvigorw install`

### 7.4 模拟器启动失败

**问题**: 模拟器无法启动

**解决方案**:
1. 检查内存是否充足
2. 关闭其他虚拟机软件（如 VirtualBox）
3. 重新创建模拟器
